#### **1. 核心交互：双击播放**

所有列表视图（精品、点赞、最近、最新、搜索结果）均遵循统一的核心交互模式：

- **双击即播**：在任何列表的任何视频条目（无论是缩略图还是文件名）上双击，都会立即执行以下操作：
  1. 切换至**播放器视图**。
  2. 立即开始播放被双击的视频。

#### **2. 播放列表联动机制**

当用户通过双击列表项开始播放时，该列表的当前状态（排序、筛选结果）会自动成为一个新的临时播放队列。

- **顺序播放**：当前视频播放结束后，系统会自动播放该临时播放列表中的下一个视频。
- **列表循环**：当列表中的最后一个视频播放完毕后，会自动从列表的第一个视频开始循环播放。
- **特殊规则 - 最近列表**：
  - “最近”列表是一个例外。当从该列表播放的视频结束后，系统**不会**继续播放列表中的下一项，而是**自动切换回全局随机播放模式**。

#### **3. 数据与视图的动态一致性**

为确保用户界面始终展示有效和最新的内容，所有列表在每次加载或变为可见状态时，都必须执行以下一致性检查：

- **文件存在性校验**：
  - 遍历列表中的每一个条目，检查其对应的物理视频文件是否存在于磁盘上。
  - 如果文件不存在（被删除或移动），该条目将**被静默地从当前视图中移除**，确保界面上不会出现无效选项。
- **封面加载策略**：
  - 系统按以下优先级顺序加载并显示封面：
    1. 
    2. **手动封面 ({hash}.webp)**：用户手动设置的封面，优先级最高。
    3. **默认封面 ({hash}_d.webp)**：系统自动生成的封面。
    4. **实时生成**：如果以上两种封面均不存在，系统会立即为该视频生成一个新的默认封面。
- **信息显示**：
  - 在所有采用**网格（Grid）缩略图**布局的列表中，均**不显示文件名**，以保持界面的简洁与视觉焦点。

#### **4. 状态保持与恢复**

为了提供流畅的导航体验，当用户在不同列表、播放器或设置页面之间切换时，每个列表的UI状态都应被记忆。

- 当用户离开一个列表视图后再次返回时，系统应恢复其离开前的状态，包括：
  - **滚动条位置**。
  - **已选中的项目**（如果列表支持选择操作）。
  - 此行为是在执行完第三条的“数据与视图的动态一致性”检查之后生效的。

#### **5. 视图切换时的播放器状态管理**

- **进入列表 → 暂停播放**：
  - 如果用户在视频**正在播放**时，从播放器视图切换到任何列表视图，当前视频的播放将**自动暂停**。
- **返回播放器 → 恢复状态**：
  - 如果用户只是从列表视图常规地切换回播放器视图（而非通过双击列表项），播放器将恢复到进入列表前的状态：
    - 
    - 如果之前是**播放中**，则**继续播放**。
    - 如果之前是**暂停**，则**保持暂停**。
