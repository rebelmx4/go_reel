- - - ### 1. 痛点与动机
  
      - 
      - **痛点**：
        - 
        - 用户刚下载或导入了一批新视频，想要立刻观看这批新内容。
        - 在海量视频的“随机播放”模式下，很难快速“随”到刚刚入库的新视频。
        - 文件名搜索需要用户记得新文件的名字，不够直观。
      - **动机**：
        - 
        - 提供一个基于**文件创建时间**的自动筛选器。
        - 聚焦于“鲜度”，让用户能零成本地直接消费最新入库的内容。
        - 限制数量（100个），保持列表的精简和时效性。
  
      ### 2. 功能概要
  
      - 
      - **自动聚合**：无需手动添加，软件根据文件的创建时间自动排序。
      - **容量限制**：动态截取所有视频中创建时间最近的 **100** 个视频。
      - **网格展示**：以缩略图形式直观展示。
      - **列表联动**：支持从该列表进入，按时间顺序（由新到旧）连续播放。
  
      ### 3. 数据来源与排序
  
      - 
      - **数据来源**：
        - 
        - 基于软件启动时扫描获取的 creation_time（创建时间）和 file_path。
        - 或者基于 **F5 刷新** 功能深度扫描后更新的元数据。
      - **排序逻辑**：
        - 
        - **Key**：文件创建时间（Creation Time）。
        - **Order**：倒序（Descending），即最新的在最前面。
        - **Limit**：Top 100。
    
      ### 4. UI 说明
    
      #### 4. 列表页面 (The Newest Page)
    
      - 
      - **布局模式**：**Grid 网格视图**。
      - 
        - 统一采用缩略图平铺。
  
      ### 5. 核心逻辑细节
  
      #### 5.1 与“刷新”功能的配合
    
      - 
    - “最新列表”对 **F5 刷新** 功能高度敏感。
      - 当用户放入新文件到目录后，必须重启软件或按下 F5 触发刷新，新文件被识别为“新增（Case 1）”后，才会出现在此列表中。
    - 如果是“重命名”或“移动”文件（Case 2），虽然文件路径变了，但只要文件系统的创建时间未变，它在最新列表中的位置可能不会剧烈变化
